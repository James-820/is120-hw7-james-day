# James Day's Homework 7

Sorry for no comments in my CSS. I chose not to include them for time constraints but in the future I will make a point of including them.

- Changes I made:
  - One thing I did was design for mobile-first.
  - I started with an inspection of the iPhone XR size to make sure it's all dialed in on mobile.
  - I actually ended up hiding the sidebar for mobile altogether, and chose to sacrifice that functionality for a better look and feel.
  - Also, on wider screens I decided to hardcode a width for the card elements, and also to have the flex direction as "row" with wrapping. That way, more card elements are visible at a time on a wider screen.
- Breakpoint justifications:
  - I basically started working with my iPhone size, my iPad size, and my normal computer size.
  - Had I worked on this for longer, I probably would have gone into Bootstrap and made styling for each size listed there.
  - I just picked some reasonable-looking pixel values based on the width displayed in the inspection window in Chrome.
  - Normally, [this website](https://getbootstrap.com/docs/5.0/layout/breakpoints/) is what I would use as reference for breakpoints.
- What I would have done differently:
  - One thing I would have done (and did this time) was design for mobile-first.
  - I think I would have waited to put more cards in until I had the first one all sized and displayed right.
  - That also goes for other elements, like buttons.
  - I would have made the first one, save the styling to the class, and then I could have easily made more of the same class that looked the same, instead of trying to get all of them good at once. It's just a little less chaotic that way I think, and would have made for more organized code.
